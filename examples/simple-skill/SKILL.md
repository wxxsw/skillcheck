# Simple Review Helper

## When to use

Use this skill for focused reviews of small code changes, bug fixes, and documentation edits.

## Instructions

1. Read the changed files before commenting.
2. Look for correctness issues first.
3. Check whether tests cover the changed behavior.
4. Keep comments specific and actionable.

## Examples

Ask the agent:

```txt
Review this pull request for correctness and missing tests.
```
