# Contributing Skills

SkillCheck prefers fewer, better entries.

Before submitting a repository, read [docs/curation-standard.md](curation-standard.md). The main catalog is for real skills and skill packs, not discovery-only lists.

## Entry requirements

Each submitted skill should include:

- Public repository URL
- Short human-readable summary
- Supported agent platforms
- Installation instructions
- License
- Risk notes
- At least one example task

## Not accepted in the main catalog

These can be useful for research, but they should not be submitted as catalog entries:

- pure `awesome-*` lists
- link directories
- ecosystem hubs
- store or marketplace pages
- plugin indexes
- generic resource lists
- repos where the actual skill is not inspectable

## Review principles

Reviewers should ask:

- Would I install this in my own coding agent?
- Can a user understand what it changes?
- Does it run scripts, fetch remote files, or read credentials?
- Does it work outside one vendor-specific client?
- Is the README clear enough to recover from failure?

## Badge proposal

Future repositories will be able to add:

```md
![SkillCheck](https://skillcheck.dev/badge/owner/repo)
```

The badge should only be issued from a reproducible scan.
